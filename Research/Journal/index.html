Reflection on Overuse of Overlays: How to Avoid Misusing Lightboxes

Jan 29, 2026

At first, overlays seem like an easy solution because they keep users on the same page. But after reading the Nielsen Norman Group article, I saw that this convenience can get annoying fast. An overlay is a small box that pops up over a page and dims the background. If it shows more than a quick message, it can cause issues. The “five Ws” checklist—who, what, when, where, and why—really stood out to me. It made me realize that overlays should have a clear purpose, not just be used out of habit. If users need to read a lot, scroll, bookmark, share, or use the Back button, a full page works better. The mobile examples showed this, since overlays can hide the close button or stop scrolling, which can trap users. After reading the article, I think overlays should only be used for urgent actions that the user starts, like confirming something hard to undo or collecting one key detail. For everything else, I would use in-page content so users can move around and stay in control.

Article link: Overuse of Overlays: How to Avoid Misusing Lightboxes

Reflection on Best practices for form design

Jan 19, 2026

According to the article "Best practices for form design", I have learned a lot of important practices to boost a form's usability. Few practices I found really important is minimizing the input fields, one column layout, explain any input or formatting requirements, label, placeholder text, form feedback, and accessibility. This practices were chosen from me because of specific reasons, minimizing the input fields was chosen because of its point of keeping the form short, and only asking questions that are actually needed. One columnb layout was chosen because it uses a one column layout so it is easier to scan and follow. For explain any input or formatting requirements, it gives guidance early, like formatting rules or examples, so users know what to do before submitting. Moving on, I bridged label and placeholder text together as one due to keeping labels visible and doesn't rely only on placeholder text because it disapperas when typing. Form feedback requires a specific message to show close to the field for an easy fix of error. Lastly, accessibility makes sure that it is readable and accessible such as good contrast, and clear text. The goal of this article is to teach how to make fewer mistakes and less frustration for the users.

Article link: Best practices for form design
